HOLA: 给线性注意力一个海马体

线性注意力和状态空间模型将前缀压缩为固定大小循环状态，获得O(1)内存优势的代价是记忆有损——当多个KV关联竞争时，早期事实被覆盖，针式召回（needle recall）退化。受互补学习系统理论启发，HOLA（Hippocampal Linear Attention）为线性注意力添加海马体补充：保留delta-rule状态作为压缩记忆，同时加入有界精确KV缓存，形成半参数化测试时记忆。缓存写入无需学习驱逐模块，保留预测残差大的token；解耦RMSNorm-gamma读取实现锐利检索。340M参数模型上Wikitext PPL从27.32降至22.92（-16.1%），超越全注意力Transformer++（26.88），RULER长程召回鲁棒扩展至32k tokens。

1. 论文定位

属于高效语言模型架构领域，解决线性注意力的记忆瓶颈问题

领域

线性注意力 / 状态空间模型

线性注意力（Linear Attention）和状态空间模型（SSM如Mamba）通过将前缀压缩为固定大小循环状态实现O(1)推理内存，是长上下文高效建模的主流方向。但其固定大小状态本质上是有损压缩。

§1 Introduction, 第1页
核心问题

为什么线性注意力"健忘"？

Delta-rule更新机制在新KV对到来时覆盖旧关联。当多个键值对竞争有限的状态容量时，早期事实被渐进擦除，导致"大海捞针"式检索在长上下文上严重退化。这不是参数规模问题，而是架构性记忆瓶颈。

§1-§2, 第1-3页
跨学科灵感

互补学习系统（CLS）

受脑神经科学中互补学习系统理论启发：哺乳动物大脑的新皮层（neocortex）通过缓慢学习形成压缩的结构化记忆，而海马体（hippocampus）快速存储具体情节记忆，两者互补。HOLA将这一思想映射到线性注意力架构中。

4. 方法精讲

双记忆系统的写入、读取和组合机制

Tech Point 1

Delta-rule压缩状态（新皮层）

HOLA保留线性注意力的标准delta-rule更新作为压缩记忆：S_t = S_{t-1} + β·(v_t - S_{t-1}·k_t)·k_t^T。其中S是固定大小的状态矩阵，β是学习率，k_t和v_t是当前token的键和值。这种更新天然具有"覆盖"性质：新信息渐进地写入状态，旧信息在容量不足时被擦除。

S_t = S_{t-1} + β · (v_t − S_{t-1} k_t) ⊗ k_t
§2 Background, 第2-3页
Tech Point 2

预测残差：记忆"惊奇度"

写入缓存的决策基于预测残差e_t = v_t - S_{t-1}·k_t，即当前值与状态预测之间的差异。β*||e||大意味着该token的信息不能被现有状态很好地预测/压缩，因此值得精确存储在缓存中。这是一种信息论驱动的"重要性"度量。

§3.2, 第4-5页
Formula

缓存写入准则

write_score_t = β_t · ‖e_t‖ = β_t · ‖v_t − S_{t-1} k_t‖

if write_score_t > τ and |cache| < B:
cache.append( (k_t, v_t) )
else if write_score_t > min_score in cache:
evict min_score token, insert (k_t, v_t)

当缓存未满时直接写入；当缓存满时，驱逐残差最小的token（最"不意外"的、已被状态吸收的信息）。阈值τ和缓存大小B是超参数。

§3.2, Algorithm 1, 第5页
Tech Point 3

为什么不用学习驱逐？

学习驱逐模块（如可训练的gating网络）增加训练复杂度和不稳定性。预测残差是delta-rule更新的"免费副产品"——它直接度量了信息被状态压缩的程度，天然适合作为缓存重要性的判据，无需额外参数。

§3.2, 第4-5页
Tech Point 4

解耦RMSNorm-gamma读取

标准做法是将缓存注意力输出y_cache与状态输出y_state直接相加，但这导致两者在同一数值尺度上"软平均"，缓存的精确信息被状态的大容量输出稀释。HOLA解耦RMSNorm的gamma缩放参数，使缓存输出有独立的缩放通路，实现锐利检索。

§3.3, 第5-6页
Tech Point 5

半参数化输出组合

最终输出为：y = RMSNorm_γ_state(y_state) + RMSNorm_γ_cache(y_cache)。其中y_state = S·q（状态检索，O(1)），y_cache = Attention(q, K_cache, V_cache)（缓存精确注意力，O(B)，B为缓存大小）。两个通路有独立的RMSNorm gamma参数，梯度可以各自流动。

y = γ_state ⊙ RMSNorm(S q) + γ_cache ⊙ RMSNorm(Attention(q, K_cache, V_cache))
§3.3 Eq.3-4, 第6页
Tech Point 6

有界缓存O(B)开销

缓存大小B是固定超参数（如1024或2048个token），因此推理时缓存注意力的计算量是常数O(B)，不随序列长度增长。这与全注意力的O(n²)形成鲜明对比，保持了线性注意力的高效性。

§3.4, 第6页
Tech Point 7

训练目标不变

HOLA不引入新的训练目标或辅助损失。缓存写入策略完全基于前向传播中的预测残差（无需梯度），模型使用标准下一个token预测损失端到端训练。缓存机制在训练和推理时均被激活。

§3.4, 第6页

每个查询token：(1)通过状态矩阵S进行O(1)检索得到y_state（压缩记忆）；(2)对有界缓存B进行精确注意力得到y_cache（海马体记忆）；(3)两个通路经各自RMSNorm缩放后相加。缓存按残差准则动态更新。

6. 核心概念可视化

关键概念的示意图解（使用AI生成示意图辅助理解）

概念图 1 · 双记忆架构

HOLA架构：新皮层压缩状态 + 海马体精确缓存

看什么

左侧"新皮层"通路：输入token经delta-rule更新固定大小状态矩阵S，提供压缩记忆（O(1)检索）。右侧"海马体"通路：预测残差大的token被写入有界KV缓存，提供精确记忆（O(B)检索）。底部两个通路经解耦RMSNorm后组合输出。

说明什么

这是互补学习系统理论在神经网络架构中的直接映射。状态S对应新皮层——缓慢学习统计规律，泛化能力强但细节有损；缓存对应海马体——快速存储具体事件，精确但容量有限。两者不是替代关系而是互补。

不要误读

缓存不是"全注意力的后门"——缓存大小B是固定常数（如1024），不随序列长度增长，因此整体复杂度仍为O(n)而非O(n²)。缓存存储的是经过残差筛选的重要token，不是全部历史。

概念示意图，对应 §3 Figure 1 架构图
概念图 2 · 覆盖问题

Delta-rule状态覆盖：旧记忆如何被新信息擦除

看什么

固定大小的状态矩阵S像一块有限大小的黑板：新KV对通过delta-rule写入时，如果状态容量已饱和，旧的键值关联会被覆盖。随着序列增长，早期token的信息被渐进擦除，导致"大海捞针"失败。

说明什么

这不是bug而是delta-rule的设计特性——它本质上是一个在线学习器，自适应地跟踪输入分布。但当需要同时记住多个独立事实时（如文档开头的随机ID号），覆盖机制就成了致命缺陷。

不要误读

覆盖不是均匀的——与当前上下文高度相关的信息更容易被保留，无关信息更快被擦除。这导致了"新近性偏差"（recency bias）：模型更容易记住最近的内容，遗忘较早的内容。

概念示意图，对应 §2 问题分析
概念图 3 · 预测残差

预测残差β*||e||：哪些token值得被海马体记住？

看什么

常见词（如"the"、"is"）的预测残差小——状态已很好地学会它们的模式，不需要缓存。罕见实体、随机ID、关键事实的预测残差大——状态未能有效压缩这些信息，需要精确存入缓存。

说明什么

β*||e||本质上度量了"这个token给状态带来了多少新信息"。大残差=信息论上的"惊喜"（surprise），意味着现有压缩记忆未能捕获该信息。将这些"惊喜"token存入缓存，恰好弥补了压缩状态的盲区。

不要误读

残差大不代表token"重要"（如噪声也可能引起大残差）。但在自然语言中，大残差通常对应有意义的不可预测信息（如实体名、数字、关键论断），这也是为什么这个简单判据在实践中有效的原因。

概念示意图，对应 §3.2 写入策略
概念图 4 · 锐利检索

解耦RMSNorm-gamma：为什么直接相加会导致模糊检索

看什么

左图（直接相加）：y_cache（精确但小）与y_state（模糊但大）直接相加，缓存信号被状态的大量级输出淹没，结果接近纯状态输出——检索"软"而模糊。右图（解耦gamma）：两个通路有独立缩放γ_cache和γ_state，缓存输出可以在需要时被放大，实现锐利检索。

说明什么

这是一个精妙的工程洞见：简单的残差连接在数值尺度不匹配时失效。y_state来自大状态矩阵的乘积，量级远大于缓存注意力输出。解耦RMSNorm归一化后用独立gamma缩放，让模型学会何时信任状态、何时信任缓存。

不要误读

γ_cache不是手动设置的——它通过反向传播学习，模型自动学会在不同上下文位置动态调整两个通路的相对权重。这比简单的残差连接更灵活。

8. 难点、误区与失败模式

容易踩的坑和方法边界

Failure Mode 1

误区：缓存越大越好

不完全正确。增大缓存B确实提升性能，但存在边际递减，且过大的B增加推理开销。关键不是缓存大小，而是写入策略是否选对了token——一个好的选择策略用小缓存就能匹敌大缓存。

Table 2 缓存大小消融, 第8页
Failure Mode 2

误区：直接相加就够了

错。如果将缓存输出直接加到状态输出而不解耦RMSNorm，缓存信号因数值量级小被状态输出"淹没"，检索效果接近纯状态模型。解耦gamma是必要的工程细节。

§3.3 + Table 2 消融, 第5-6,8页
Failure Mode 3

误区：新近性缓存足够

只缓存最近B个token（recency cache）不能解决早期信息遗忘问题。RULER实验显示recency cache在长上下文检索上远不如残差驱动的写入策略，因为重要信息可能出现在上下文任何位置。

§4.2 Figure 3, 第9页
Limitation 1

规模验证有限

所有实验在340M参数规模完成。HOLA在更大规模（如7B、70B）是否同样有效？gamma解耦和残差写入策略是否需要随规模调整？这些问题尚未验证。

§5 局限性讨论, 第11页
Limitation 2

缓存驱逐是贪心的

当前驱逐策略只看单个token的残差分数，不考虑token间的协同/冗余关系。两个中等残差的token可能组合起来携带关键信息，但贪心驱逐可能逐个移除它们。全局最优的缓存选择是未来方向。

终极总结

HOLA受互补学习系统理论启发，为线性注意力架构添加了一个"海马体"——有界精确KV缓存。通过基于预测残差β*||e||的无学习写入策略，自动选择"状态未能压缩的惊奇信息"存入缓存；通过解耦RMSNorm-gamma的读取机制，实现缓存与状态的锐利组合而非模糊平均。在340M参数规模上，HOLA将Wikitext PPL降低16.1%至22.92，超越全注意力Transformer++，并在32k tokens（训练长度16倍）的RULER长程召回上保持鲁棒。核心洞见是：线性注意力的"健忘"不是容量问题而是架构问题——给压缩记忆补充一个小而精确的"海马体"缓存，就能在保持O(1)+O(B)效率的同时获得接近完美的长程记忆能力。
