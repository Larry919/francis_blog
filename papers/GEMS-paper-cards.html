GEMS: 几何约束实现大模型多语义叠加

现有激活引导（Activation Steering）只能单一方向控制LLM行为，多方向叠加即崩溃。GEMS揭示崩溃的两个独立根源——分布偏移与方向干扰，并通过纯几何约束（范数保持+实时正交化+o_proj精确注入+高斯包络）实现免训练的多语义并行引导，GSM8K三方向注入从4%崩溃恢复至98%准确率。

1. 论文定位

属于LLM可解释性与模型控制领域，解决多方向激活引导的崩溃问题

领域

LLM激活引导 / 可解释性

Activation Steering通过推理时修改中间隐藏状态控制模型行为，无需微调。现有方法（ActAdd、RepE、ITI、CAA）均局限于单一方向控制，多方向叠加即崩溃。

为什么多方向注入会崩溃？

此前认为崩溃是激活引导范式的固有局限，多方向控制必须训练。GEMS证明崩溃有明确几何结构，可以通过纯前向传播约束解决。

无需修改模型权重或引入优化步骤，纯几何操作即可在3B-31B模型上迁移。四个约束：范数保持、o_proj注入、实时正交化、高斯包络。

4. 方法精讲

四个几何约束的技术细节与数学原理

Tech Point 1

对比专家向量提取

对每个语义方向定义一对对比prompt：base prompt是中性任务查询，expert prompt是表达特定语义立场的陈述句。第l层的专家方向 = Normalize(mean-pooled(h_expert) - mean-pooled(h_base))，排除首尾special token以避免边界异常。

v_i^(l) = Normalize( h̄_expert_i^(l) − h̄_base^(l) )
§3.1, 第4-5页
Tech Point 2

实时Gram-Schmidt正交化

每个token在每个hook层，先计算当前注意力方向u_δ，然后依次将每个专家向量投影到{u_δ, e_1,...,e_{k-1}}的正交补空间。

§3.2 Eq.1-2, 第5页
Formula

正交化公式

u_δ = ΔH / ‖ΔH‖ （当前注意力方向单位化）
e_k = Normalize( v_k − (v_k·u_δ)·u_δ − Σ_{j=1}^{k-1} (v_k·e_j)·e_j )

确保每个e_k与u_δ和所有之前的e_j正交，消除方向间的线性依赖。专家顺序可任意调整。

Eq.1-2, 第5页
Tech Point 3

范数保持加权叠加

H_target = ‖ΔH‖ · (w_base·u_δ + Σw_i·e_i)，权重满足 w²_base + Σw²_i = 1。严格保持输出范数等于原始注意力输出范数。

§3.3, 第5-6页
Tech Point 4

o_proj精确注入点

Hook只拦截注意力输出投影(o_proj)，MLP通路完全不触碰：H_new = H_old + ΔH_mlp + H_out(ΔH_attn)。MLP编码事实知识，污染它立即崩溃。

§3.3 Eq.3, 第6页; Table 3 T8, 第8页
Tech Point 5

高斯包络层间调制

E(l) = exp(−(l−μ)²/(2σ²))，集中干预于中间层（μ=14, σ=3），首尾层平滑衰减。L16-L20额外余弦衰减τ(l)。

§3.4 Eq.4, 第5-6页
Tech Point 6

层间插值与部署

每层输出为原始激活与引导激活的插值：H_out = Normalize((1−E(l))·ΔH + E(l)·H_target) · ‖ΔH‖。GEMS在ActAdd基础上hook前向传播，无需修改模型图或添加训练参数。默认参数：干预层L9-L20（12层），强度[0.12, 0.08, 0.04]。

§3.4 + Appendix C, 第6页
Tech Point 7

为什么o_proj？

注意力输出承载上下文关系信息，MLP承载事实知识。在o_proj处注入可以引导注意力模式而不破坏知识库，这是论文最反直觉但最有效的发现。

§3.3 + §4.3 T8消融, 第6,8页
Tech Point 8

语义特异性验证

每个专家方向的native词汇簇在三方向叠加后仍保持高激活（70-88%正向率，p<0.01），随机正交向量仅48%。证明效果来自语义而非几何操作本身。

Figure 1 · 整体架构

GEMS架构：三个核心几何约束的可视化

看什么

三个子图：(a)高斯包络在L14达到峰值、向L12和L16衰减；(b)Hook精确拦截Self-Attn的o_proj输出ΔH_attn，MLP通路ΔH_mlp完全不被触碰；(c)原始专家向量v_i（浅红色）高度共线（cos 0.74-0.87），正交化后的e_i（红色）方向分离，约束在单位球内w²_base+Σw²_i=1。

说明什么

这是GEMS方法的全景图。三个约束如何协同：高斯包络控制"在哪几层注入多强"，o_proj Hook控制"在什么位置注入"，正交化+范数保持控制"注入什么方向和多大强度"。

不要误读

子图(c)中的"Raw Experts (v_i, interfering)"不是说原始向量是错误的，而是说它们在共享子空间高度重叠，直接相加会导致信号抵消。正交化不是改变语义，而是将它们分解到独立方向。

Figure 2 · 诊断证据

分布偏移：残差流范数在干预窗口内飙升3.4倍

看什么

蓝色Baseline曲线从L0(2.1)平滑增长到L31(84.7)。红色ActAdd曲线在干预窗口L9-L20（阴影区域）内急剧加速上升，在L18-L20达到基线的3.4倍，然后持续高位传播到后续层。

说明什么

加性注入在多层累积导致范数爆炸，将激活值推出模型训练分布——这是"分布偏移"崩溃源的直接可视化证据。基线的平滑范数增长是训练期间学到的稳定模式。

不要误读

不是范数大就一定崩溃，关键是偏离了训练时的范数增长轨迹。LayerNorm虽然归一化但残差连接使扰动仍可传播。

Figure 3 · 诊断证据

概率分布崩溃：Top-1从43%降到11%，熵翻倍

这不是"多样性增加"的正面效果——看定性输出（Table 1），ActAdd产生的是重复无意义内容（"self-moralizing certainties of self-vulnerability"），而非多样化的有意义输出。

方向干扰：并行注入时各方向对齐度下降34-43%

看什么

这不是"注入强度不够"——每个方向在并行条件下用的强度和单独时一样(α=0.5 each)。问题在于向量几何，不是力度。

8. 难点、误区与失败模式

容易踩的坑和方法边界

Failure Mode 1

误区：正交化是核心约束

错。去正交化(T3)仍有92%准确率（等于基线）。正交化解决的是"语义可区分性"而非"会不会崩"。真正的生死线是范数保持+o_proj注入。

Table 3 T3 vs T6/T8, 第8页
Failure Mode 2

误区：注入点不重要

错。T8仅将注入点从o_proj改为full layer，其他约束全保留，准确率从98%→4%。MLP编码事实知识，污染它即摧毁推理能力。

Table 3 T8, 第8页
Failure Mode 3

分布外向量导致崩溃

即使满足所有几何约束，如果专家向量不在模型学到的表示空间内（如随机向量），仍可能崩溃。向量必须来自模型自身的对比激活差异。

§3.1警告 + §5讨论, 第5,11页
Limitation 1

仅验证3个方向

所有实验最多3个同时注入的方向。方向数增加时（如10+），正交化是否仍然有效、范数预算如何分配，尚未验证。

§5局限性, 第11页
Limitation 2

不能创造新能力

GEMS只是在模型已有能力空间内重新加权输出分布（词汇簇分析证实了这一点），不能赋予模型训练数据中没有的能力。

终极总结

GEMS通过"范数保持+o_proj精确注入+实时Gram-Schmidt正交化+高斯包络层间调制"四个纯几何约束，首次证明了免训练的LLM多语义并行激活引导是可行的。它将三方向叠加在GSM8K上从4%崩溃恢复到98%准确率，Wikitext-2 PPL仅增2.2%，效果可跨3B-31B架构迁移。核心洞见是：崩溃不是激活引导的固有局限，而是无约束加法违反了表示空间几何的必然结果——只要尊重几何结构，一个模型可以同时被多个语义方向精确引导。
